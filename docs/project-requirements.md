# Fridge Recipe 프로젝트 요구사항 정의서 (PRD)

## 1. 프로젝트명

- Fridge Recipe
- 유통기한 기반 식재료 우선소비 및 맞춤형 레시피 추천 웹앱

## 2. 프로젝트 목적

- 냉장고 속 식재료의 유통기한을 관리하고, 버려지는 재료를 줄이는 것을 목표로 합니다.
- 보유 재료를 기준으로 만들 수 있는 레시피를 추천합니다.
- 부족 재료를 함께 안내하여 실제 조리 가능성을 높입니다.

## 3. 문제 정의

- 냉장고 속 재료를 잊어버려 유통기한이 지나 버리는 문제
- 어떤 재료를 먼저 소비해야 하는지 파악하기 어려운 문제
- 보유 재료로 만들 수 있는 요리를 바로 떠올리기 어려운 문제
- 레시피를 찾더라도 추가 구매해야 하는 재료를 한눈에 파악하기 어려운 문제

## 4. 주요 사용자

- 냉장고 속 식재료를 직접 관리하는 사용자
- 자취생 또는 1인 가구
- 장보기 전 보유 재료를 확인하고 싶은 사용자
- 유통기한 임박 재료를 먼저 소비하고 싶은 사용자

## 5. 핵심 기능 요구사항

- 사용자는 식재료명, 수량, 유통기한을 입력해 재료를 등록할 수 있어야 합니다.
- 사용자는 등록된 식재료 목록을 확인할 수 있어야 합니다.
- 사용자는 등록된 식재료를 수정하거나 삭제할 수 있어야 합니다.
- 시스템은 유통기한을 기준으로 D-day를 계산해야 합니다.
- 시스템은 소비 기한이 임박한 식재료를 별도 영역에 표시해야 합니다.
- 시스템은 보유 재료를 기준으로 추천 레시피를 제공해야 합니다.
- 시스템은 기한 만료 재료를 추천 계산에서 제외해야 합니다.
- 시스템은 추천 레시피에 필요한 부족 재료를 표시해야 합니다.
- 시스템은 부족 재료가 3개 이하인 레시피만 추천해야 합니다.
- 시스템은 추천 점수를 100점 만점 기준으로 표시해야 합니다.
- 사용자는 레시피 상세보기 모달에서 간단 조리 순서를 확인할 수 있어야 합니다.

## 6. 비기능 요구사항

- 별도 설치 없이 브라우저에서 실행 가능해야 합니다.
- 화면 구성이 직관적이어야 합니다.
- 식재료 등록, 수정, 삭제 후 추천 결과가 즉시 갱신되어야 합니다.
- 새로고침 후에도 같은 브라우저에서는 데이터가 유지되어야 합니다.
- 외부 레시피 사이트의 콘텐츠를 무단으로 사용하지 않아야 합니다.

## 7. 데이터 저장 방식

- 사용자 식재료 데이터는 브라우저 localStorage에 저장합니다.
- 레시피 데이터는 recipeData.js의 RECIPE_DATABASE에서 관리합니다.
- 현재 버전은 서버 DB와 로그인 기능이 없는 MVP 구조입니다.

## 8. 추천 기준

- 보유 재료와 레시피 ingredients 비교
- 기한 만료 재료 추천 제외
- 오늘까지, D-1~D-3 재료 우선소비 가중치 적용
- 부족 재료 3개 이하 조건 적용
- 추천 점수는 조리 적합도를 나타내는 100점 만점 기준

## 9. 현재 구현 범위

- 식재료 등록, 수정, 삭제
- D-day 계산
- 소비 기한 임박 표시
- 추천 레시피 출력
- 부족 재료 표시
- 추천 점수 표시
- 레시피 상세보기 모달
- localStorage 저장

## 10. 현재 한계

- 로그인 기능 없음
- 실제 서버 DB 없음
- 다른 기기 간 데이터 동기화 불가
- 레시피 데이터에 없는 재료는 추천 레시피로 연결되지 않을 수 있음

## 11. 향후 확장 방향

- 로그인 기능 추가
- 사용자별 냉장고 데이터 DB 저장
- 추천 이력 저장
- 장보기 리스트 자동 생성
- 레시피 데이터베이스 확장
- 사용자 소비 패턴 기반 개인화 추천
