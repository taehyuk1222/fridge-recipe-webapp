# Fridge Recipe 사용 가이드

## 1. 실행 방법

- GitHub 저장소 다운로드
- 다운로드한 ZIP 압축 해제
- 폴더 내의 index.html 실행

주의사항: 정상적인 실행을 위해서는 index.html 단독 실행이 아닌, style.css, script.js, recipeData.js, recommend_logic.js, recommend_sort.js 파일이 모두 동일한 폴더 내에 위치해야 합니다.

## 2. 식재료 등록 방법

- 재료명 입력: 등록할 식재료의 이름을 입력합니다.
- 수량 입력: 보관 중인 수량을 입력합니다.
- 유통기한 선택: 달력에서 해당 식재료의 유통기한을 지정합니다.
- 등록하기 버튼 클릭: 정보를 모두 입력한 후 등록 버튼을 누릅니다.
- 등록된 식재료는 화면의 나의 냉장고 보관함 목록에 즉시 표시됩니다.

## 3. 냉장고 보관함 확인

- 나의 냉장고 보관함에서 등록한 재료명, 수량, 유통기한, D-day 상태를 확인할 수 있습니다.
- 브라우저를 새로고침하더라도 localStorage를 통해 데이터가 그대로 유지됩니다.

## 4. 식재료 수정 및 삭제

- 등록된 각 식재료 항목에서 수정 및 삭제 버튼을 클릭하여 정보를 변경하거나 지울 수 있습니다.
- 식재료를 삭제하거나 수정하면 냉장고 목록뿐만 아니라 추천 레시피 결과도 함께 갱신됩니다.

## 5. 소비 기한 임박 식재료 확인

- 기한 만료, 오늘까지, D-1 ~ D-3에 해당하는 재료는 소비 기한 임박 영역에서 따로 확인할 수 있습니다.
- 기한 만료 재료는 임박 영역에 표시되지만, 추천 계산 과정에서는 제외됩니다.

## 6. 추천 레시피 확인

- 보유 재료를 기준으로 조리 가능한 추천 레시피가 동적으로 출력됩니다.
- 추천 레시피 카드에는 보유 중인 재료와 추가 구매가 필요한 부족 재료가 구분되어 표시됩니다.
- 요리를 위해 추가로 필요한 부족 재료가 3개 이하인 레시피만 추천 목록에 나타납니다.

## 7. 추천 점수 확인

- 각 추천 레시피에는 100점 만점 기준의 추천 점수가 표시됩니다.
- 이는 현재 냉장고 상태에서 해당 레시피가 얼마나 만들기 적합한지 보여주는 지표입니다.
- 보유 재료 일치도, 우선소비 재료 포함 여부, 부족 재료 수를 기준으로 계산됩니다.

## 8. 레시피 상세보기

- 추천 카드의 레시피 상세보기 버튼을 클릭하면 중앙 모달 창이 열립니다.
- 모달 내에서 레시피명, 추천 이유, 보유 재료, 부족 재료, 조리 시간, 난이도, 추천 점수, 간단 조리 순서 등을 확인할 수 있습니다.
- 모달 창은 닫기 버튼 클릭, 바깥 영역 클릭, 또는 키보드 ESC 키를 눌러 닫을 수 있습니다.

## 9. 데이터 저장 방식

- 현재 버전은 서버 DB가 아닌 사용자의 브라우저 localStorage에 데이터를 저장합니다.
- 같은 브라우저 내에서는 데이터가 유지됩니다.
- 단, 다른 기기나 다른 브라우저 간에는 데이터가 공유되지 않습니다.

## 10. 사용 시 주의사항

- index.html 파일만 단독으로 실행하지 마십시오.
- 폴더 구조를 유지한 상태로 실행해야 정상 작동합니다.
- 브라우저의 localStorage(인터넷 사용 기록 등)를 삭제하면 등록한 식재료 데이터도 함께 삭제될 수 있습니다.
