# Fridge Recipe 추천 알고리즘

## 1. 추천 기능 개요

- 사용자가 등록한 냉장고 식재료를 기준으로 추천 레시피를 계산합니다.
- 단순히 보유 재료가 포함된 모든 요리를 추천하는 것이 아니라, 유통기한과 부족 재료 수를 함께 고려합니다.
- 핵심 목적은 유통기한이 임박한 재료를 우선 소비하도록 돕는 것입니다.

## 2. 사용 데이터

- 사용자 식재료 데이터는 localStorage에 저장된 냉장고 재료 목록을 사용합니다.
- 레시피 데이터는 recipeData.js의 RECIPE_DATABASE를 사용합니다.
- 각 레시피는 name, ingredients, needed, cookingTime, difficulty, category, steps 정보를 가집니다.

## 3. 추천 후보 선정 기준

- 사용자가 보유한 재료와 레시피 ingredients가 최소 1개 이상 일치해야 합니다.
- 기한 만료 재료는 추천 계산에서 제외합니다.
- 부족 재료가 3개 이하인 레시피만 추천 후보로 선정합니다.
- 부족 재료가 4개 이상이면 현실적으로 바로 만들기 어렵다고 판단해 추천에서 제외합니다.

## 4. 기한 만료 재료 제외

- 기한이 지난 재료는 소비 기한 임박 영역에는 표시되지만, 추천 계산에는 사용하지 않습니다.
- 예를 들어 콩나물이 기한 만료 상태이면 콩나물국 추천에 콩나물을 활용 재료로 사용하지 않습니다.
- 이는 상한 재료를 활용하도록 안내하는 문제를 방지하기 위한 기준입니다.

## 5. 우선소비 가중치

- 오늘까지인 재료나 D-1~D-3 재료가 포함된 레시피는 추천 우선순위를 높입니다.
- 유통기한이 가까운 재료를 먼저 소비하도록 유도하기 위한 기준입니다.
- D-4 이상 재료는 일반 보유 재료로 계산합니다.

## 6. 부족 재료 기준

- 추천 카드에는 보유 중인 재료와 추가 구매 필요 재료를 구분해 표시합니다.
- 부족 재료가 적을수록 추천 점수가 높아집니다.
- 부족 재료 0개는 바로 요리 가능 상태입니다.
- 부족 재료 1~3개는 추가 구매 후 조리 가능 상태입니다.
- 부족 재료 4개 이상은 추천 결과에서 제외합니다.

## 7. 추천 점수 기준

- 추천 점수는 100점 만점 기준입니다.
- 현재 냉장고 상태에서 해당 레시피가 얼마나 만들기 적합한지를 나타냅니다.
- 추천 후보로 선정된 레시피는 50~100점 범위로 표시합니다.
- 점수는 보유 재료 일치도, 우선소비 재료 포함 여부, 부족 재료 개수를 기준으로 계산합니다.

## 8. 추천 점수 해석

- 점수가 높을수록 현재 보유 재료로 만들기 쉽고, 유통기한이 임박한 재료를 효과적으로 활용할 수 있습니다.
- 점수는 영양학적 점수나 맛 평가가 아니라 조리 적합도 점수입니다.

## 9. 추천 결과 갱신 방식

- 식재료를 등록하면 추천 결과가 다시 계산됩니다.
- 식재료를 수정하면 D-day와 추천 결과가 함께 갱신됩니다.
- 식재료를 삭제하면 삭제된 재료를 제외하고 추천 결과가 다시 계산됩니다.
- 새로고침 후에도 localStorage에 저장된 데이터를 기준으로 추천이 유지됩니다.

## 10. 현재 한계

- 레시피 데이터에 없는 재료는 추천 레시피로 연결되지 않을 수 있습니다.
- 실제 서버 DB나 로그인 기능은 아직 없습니다.
- 현재는 브라우저 localStorage 기반 MVP 구조입니다.

## 11. 향후 개선 방향

- 레시피 데이터베이스 확장
- 사용자별 냉장고 데이터 DB 저장
- 추천 이력 저장
- 자주 부족한 재료 분석
- 사용자 소비 패턴 기반 개인화 추천
