# Fridge Recipe

유통기한 기반 식재료 우선소비 및 맞춤형 레시피 추천 웹앱

## 1. 프로젝트 개요

Fridge Recipe는 냉장고 속 식재료의 유통기한을 관리하고, 보유 재료를 기준으로 만들 수 있는 레시피를 추천하는 웹앱입니다. 사용자가 식재료를 등록하면 D-day를 계산하고, 소비 기한이 임박한 재료를 먼저 소비할 수 있도록 유도합니다.

사용자의 냉장고 상태를 기반으로 현재 조리 가능한 레시피를 추천하며, 추천된 레시피를 만들기 위해 추가로 필요한 재료를 함께 안내합니다. 이를 통해 버려지는 식재료를 줄이고 보다 효율적으로 식사를 준비할 수 있습니다.

## 2. 개발 배경

- 냉장고 속 재료의 존재를 잊어 유통기한이 지나 버리게 되는 문제 해결
- 현재 보유한 재료만으로 어떤 요리를 할 수 있는지 파악하기 어려운 점 개선
- 특정 요리를 시도하려 할 때 부족한 재료를 미리 확인하지 못해 조리를 포기하게 되는 상황 방지

## 3. 주요 기능

- 식재료 등록
- 식재료 목록 출력
- 식재료 수정 및 삭제
- 유통기한 D-day 계산
- 소비 기한 임박 식재료 표시
- 보유 재료 기반 추천 레시피 제공
- 레시피 상세보기 모달
- 부족 재료 및 추가 구매 재료 표시
- localStorage 기반 데이터 저장

## 4. 추천 알고리즘 상세

본 프로젝트의 레시피 추천은 다음 기준을 통해 산출됩니다.

- 사용자의 보유 재료와 각 레시피의 요구 재료를 비교 분석합니다.
- 위생 및 안전을 위해 유통기한이 지난 식재료는 조리 가능 재료에서 제외합니다.
- 유통기한이 오늘까지이거나 1~3일(D-1 ~ D-3) 남은 임박 식재료가 포함된 레시피에는 가중치를 부여해 우선적으로 추천합니다.
- 요리를 위해 추가 구매해야 하는 부족 재료가 3개 이하인 레시피만 추천 목록에 포함합니다.
- 현재 냉장고 상태를 기준으로 해당 레시피의 조리 적합도 및 임박 재료 포함 여부를 종합하여 100점 만점의 추천 점수를 산출합니다.
- 최종 추천 후보로 선정된 레시피는 50점부터 100점 사이의 점수 범위로 화면에 표시됩니다.

## 5. 실행 방법

프로젝트를 로컬 환경에서 실행하는 방법은 다음과 같습니다.

1. 본 GitHub 저장소를 다운로드합니다.
2. 다운로드한 ZIP 파일의 압축을 해제합니다.
3. 폴더 내의 index.html 파일을 웹 브라우저로 실행합니다.

주의사항: 정상적인 실행을 위해서는 index.html 단독 실행이 아닌, style.css, script.js, recipeData.js, recommend_logic.js, recommend_sort.js 파일이 모두 동일한 폴더 내에 위치해야 합니다.

## 7. GitHub 협업 과정

본 프로젝트는 팀원 간 협업과 소스코드 관리를 위해 GitHub 기능을 활용했습니다.

- 구현 기능별로 Issue를 생성하여 작업 범위를 분리했습니다.
- 브랜치를 생성해 작업 내용을 안전하게 분리하고 충돌을 방지했습니다.
- 작업이 완료된 브랜치는 Pull Request를 통해 main 브랜치에 병합했습니다.
- Issue #15에서 팀원이 작성한 추천 기능 초안을 검토한 뒤, 실제 웹앱 화면과 동적으로 연결하고 세부 로직을 고도화했습니다.
- 이후 #6 ~ #8에 할당된 추천 관련 이슈들이 정상적으로 반영되었음을 확인하고 close 처리했습니다.

## 9. 현재 버전의 한계

- 서버를 거치지 않고 브라우저의 localStorage를 활용하는 MVP 단계입니다.
- 실제 서버 데이터베이스 및 사용자 로그인 기능이 없습니다.
- 로컬 스토리지 특성상 다른 기기 간의 데이터 동기화 및 공유가 불가능합니다.
- 시스템에 정의된 레시피 데이터에 없는 재료를 등록할 경우, 해당 재료는 추천 레시피 계산 과정에 반영되지 않을 수 있습니다.

## 10. 향후 확장 방향

- 다양한 요리를 지원하기 위한 대규모 레시피 데이터베이스 확장
- 사용자 소비 패턴을 기반으로 한 개인화 맞춤형 추천 알고리즘 도입
